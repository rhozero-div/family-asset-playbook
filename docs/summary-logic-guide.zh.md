# 综合建议摘要逻辑说明

这份文档面向顾问，目的是解释剧本中的**综合建议摘要**是怎么生成的，方便顾问：

1. 自己理解这一部分在做什么
2. 向客户解释为什么系统会给出这些结论
3. 分清哪些是方法论逻辑，哪些是表达层的通俗翻译

English version: [summary-logic-guide.md](summary-logic-guide.md)

## 1. 这部分到底是做什么的

综合建议摘要不是一套独立于剧本之外的第二模型。
它只是把整份剧本里最值得先看的结论，压缩成前置阅读层。

它当前主要回答五类问题：

1. 已录入的重大节点按当前资料能不能覆盖
2. 如果存在富余资金账户，它的长期持有区间大致如何
3. 当前家庭处在哪个阶段，优先级最高的动作是什么
4. 保障结构有没有明显缺口
5. 哪些信号要盯，哪些变化出现时要重算

## 2. 它用的是已有结果，不是另起一套判断

综合建议摘要读取的是整份剧本已经算出来的结果，包括：

- 客户档案
- 重大节点推演结果
- 逐年现金流推演
- 资金分层计划
- 带收益的 bucket 推演
- 保障结构分析

所以摘要如果变化，通常意味着底层输入或底层计算发生了变化。

## 3. 最高层结论

当前摘要最前面会给两个高层结论。

### 3.1 重大节点覆盖结论

这部分回答：

- 在当前资料和假设下，已录入的重大节点是否都能覆盖

如果全部可覆盖：

- 会明确说“当前节点整体可覆盖”
- 通常还会提最后一个重大节点之后还剩多少资产

如果存在缺口：

- 会指出最早出现缺口的节点
- 给出大致缺口规模

之所以放在最前面，是因为客户最关心的通常就是：这份规划到底顶不顶得住这些人生节点。

### 3.2 富余资金长期收益结论

这一条只在单独存在富余资金账户时出现。

它回答：

- 如果把富余资金当作长期资金持有到当前测算终点，大致会落在怎样的年化增长区间

当前表达方式通常包括：

- 居中结果
- 偏弱结果区间
- 偏顺利结果区间

它的作用不是承诺收益，而是把“近钱”和“长钱”区分开来。

## 4. 当前阶段判断

高层结论之后，摘要会给出“当前家庭处于什么阶段”的判断。

这不是一个单独评分模型，而是从几类信号里选出当前最重要的状态，例如：

- 当前月度结余是否为负
- 最近有没有提前出现的缺口
- 重大节点是否已经临近
- 退休后的现金流压力是否提前逼近
- 是否已经存在明显的富余资金账户

当前常见的阶段判断包括：

- 现金流修复优先
- 目标取舍与预算重排优先
- 近期节点保障优先
- 退休准备前置
- 分层后可持续增值
- 中期积累推进

它的作用，是让客户先明白“这份剧本最核心的问题是什么”。

## 5. 当前最值得优先做的事

接下来摘要会从底层结果里抽取最有行动性的优先事项。

它主要会结合这些因素：

- 当前月度结余
- 应急层是否达标
- 最近未达标 bucket
- 最近重大节点
- 退休缺口
- 是否已经存在富余资金账户

常见动作包括：

- 先修复现金流
- 先补足应急层
- 把最近节点资金单独锁定
- 按节奏补最近未达标账户
- 把长钱和近钱分开
- 提前为退休后现金流做承接准备

这些都不是产品建议，而是规划优先级建议。

## 6. 保险配置建议

当前保险部分的口径是**结构缺失分析**，不是完整保险规划。

原因很简单：当前问卷并没有收集足够做完整保险推荐的输入，例如：

- 当前总保费压力
- 具体保单条款
- 不同保额对应的真实保费成本

所以当前摘要只聚焦于结构层面，例如：

- 医疗险结构是否缺
- 重疾保障是否明显不足
- 核心收入者是否缺定寿

同时它还会考虑现金流压力：

- 如果当前现金流已经偏紧，不会暗示“一次性全面补齐”
- 更适合先讲优先顺序和保费承受力

## 7. 后续需要持续关注的信号

摘要还会给出一组“值得持续盯”的信号。

常见包括：

- 月度净结余是否跌破当前基线
- 应急层是否仍保持独立
- 最近未达标 bucket 是否按节奏推进
- 最紧年份的常规净现金流
- 负债月供占收入比
- 退休后月度缺口
- 富余资金是否被提前挪用

这样做的目的，是让客户知道该盯哪几个指标，而不是盯全部数字。

## 8. 什么时候应该重算

摘要最后会列出触发重算的变化。

当前主要围绕真实生活变化，而不是日常市场波动，例如：

- 收入明显变化
- 支出或负债月供明显变化
- 重大节点新增、取消、提前、延后或放大
- 退休时间变化
- 子女教育路径变化
- 购房安排变化
- 应急层被长期动用
- 富余资金被提前挪用

背后的原则是：

- 对这份剧本影响最大的，通常不是每天的市场波动，而是家庭事实本身发生了变化

## 9. 为什么摘要要尽量说人话

底层是量化逻辑，但摘要不会优先使用专业术语。

例如在客户端表达时，更适合说：

- 偏弱结果区间
- 常见结果区间
- 居中结果
- 偏顺利结果区间

目的不是弱化严谨性，而是把严谨性变成客户可用的语言。

## 10. 这部分不做什么

综合建议摘要不会：

- 取代后面的详细推演表
- 取代 bucket 配置逻辑
- 给出产品推荐
- 承诺收益或承诺节点一定可达成
- 取代法律、税务、精算或保险专业意见

它只是前置结论层和行动优先级层。
