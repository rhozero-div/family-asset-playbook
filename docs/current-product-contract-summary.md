# 当前产品合同摘要

**日期:** 2026-06-28
**作用:** 作为当前版本 `family-asset-playbook` 的简明合同摘要，用于快速检查问卷、剧本、程序是否仍然一致。

---

## 1. 产品定位

- 这是一个**家庭资产 high-level 规划工具**
- **不构成投资建议**
- 顾问使用问卷采集与校正信息
- 最终客户交付物是**剧本**
- 当前主产品不是“最优权重求解器”，而是“事件驱动 + 现金流分层 + bucket 执行”的规划剧本

---

## 2. 当前主链路

1. 问卷/采集结果产出 YAML
2. loader 读取家庭、事件、收入、资产、保险、储蓄险、假设覆盖
3. projection 做事件节点推演
4. allocator 按用途切 bucket
5. markdown/html renderer 输出客户剧本

---

## 3. 输入合同

### 3.1 最小必需顶层字段

- `profile_version`
- `schema_version`
- `family`
- `events`

### 3.2 当前重要兼容/回退规则

- 若未识别到 `primary_breadwinner`，当前回退:
  - `family_name = "unknown"`
  - 主要收入者年龄按 `35` 岁推导
- `advisor_assessment.risk_tolerance` 缺失时，默认 `balanced`
- `retirement_age` 缺失时，默认 `60`
- `retirement_expense_coeff` 缺失时，默认 `0.7`
- 有医疗保障但未录入赔付率时，默认 `0.80`
- 旧聚合医疗口径缺失时，默认:
  - `starting_annual = 20000`
  - `growth_rate = 0.05`
  - `annual_cap = 80000`
- 保险缴费期(`term_life / ci / medical`)当前家庭级聚合取**成员中的最大 pay_years**

### 3.3 储蓄险合同

- `assets.financial.savings` **不计入** `total_financial_assets`
- 若 `linked_account` 匹配某个事件/心理账户，则并入该账户初始配置
- 未指定或未匹配部分并入`富余资金`

### 3.4 当前问卷结构

1. 家庭成员输入,以及人生节点相关信息
2. 收入情况
3. 常规支出情况
4. 重大支出规划
5. 现有资产
6. 现有保险
7. 风险偏好与推演假设

---

## 4. 时间与现金流合同

### 4.1 统一时点

- 重大事件统一按**事件年份年末**发生
- 未来现金流统一按**逐年累计、年末入账**
- 当前主链路**不再按月度复利/逐月事件**计算
- 所有主图表统一按 `measurement_end_year` 截断

### 4.2 测算截止年份合同

- 字段位置:`assumptions.projection.measurement_end_year`
- 当前默认值:
  - 若已录入重大节点,默认取**最后一个重大节点年份**
  - 若未录入重大节点,默认回退为 `当前年份 + 30`
- 前后端共同约束:不得早于最后一个重大节点年份

### 4.3 节点缺口测算

- `required return` 只作为沟通信号，不是收益承诺
- 求解范围固定为 `0% ~ 30%`
- 剩余年数 `<= 0` 时，直接返回 `30%`
- 若未来某年净现金流为负，在 required return 求解时按 `0` 处理，不作为负追加

### 4.4 缺口可行性分档

- `conservative`: `<= 4.5%`
- `balanced`: `<= 8.0%`
- `aggressive`: `<= 12.0%`
- `infeasible`: `> 12.0%`

---

## 5. 资产分层合同

### 5.1 应急层

- 目标优先读取 `assets.liquidity_reserve_months`
- 缺失时回退为 `6` 个月
- bucket-MC 中会自动再平衡:
  - 高于目标时释放超额
  - 低于目标时优先从富余资金补足

### 5.2 节点 bucket

- 事件按时间窗口切成阶段 bucket
- 节点权重优先读取 `assumptions.phases[].weights`
- 同时兼容 `0-1` 小数和 `0-100` 百分数写法
- 若 phase weights 缺失或异常，回退到默认阶段模板

### 5.3 富余资金

- 满足应急与已识别目标后，剩余部分进入 `富余资金`
- 当前默认权重:
  - 固收 `17.5%`
  - 权益 `62.5%`
  - 保险 `5%`
  - 另类 `15%`
- 若存在退休事件，退休年份作为其阶段切换时点

### 5.4 年度净结余分配顺序

1. 先补应急
2. 再补最早到期节点
3. 同一年内按“剩余缺口 ÷ 剩余年份”节奏分配
4. 最后才进入富余资金

---

## 6. 保险合同

- 当前剧本只做**保障结构缺失分析**
- 不做具体保险产品推荐
- 不做目标保额测算
- 不展示“加保缺口/保障总目标”图表
- validator 仍兼容部分旧保险字段，因为 loader 还保留向后解析

---

## 7. 剧本输出合同

### 7.1 当前稳定结构

1. `综合建议摘要`
2. `A. 客户情况概览`
3. `B. 资产推演`
4. `C. 资产配置执行方案`
5. `C3. 心理账户余额（居中情景）`
6. `C4. 心理账户余额(按阶段着色)`
7. `C5. 各层余额时序堆叠`
8. `C6. 各层余额与资金来源`

### 7.2 综合建议摘要当前职责

- 先给客户一个最高层结论,包括重大支出节点整体是否可覆盖,以及富余资金长期增值的大致区间如何理解
- 再总结当前家庭所处阶段与最优先动作
- 说明重大支出节点应对与未来现金流分配顺序
- 说明长期资产增值安排与复盘触发点
- 仅做保障结构缺失分析,不做产品推荐或目标保额测算

### 7.3 图表/表格口径

- bucket 图与余额图使用年末状态
- 节点 bucket 在支取年份年末清零
- 终老推演按 `5` 年一步展示
- B 段与 C 段主要图表统一展示到 `measurement_end_year`

## 8. 文档权威顺序

当前若发生冲突，按以下顺序判断:

1. 当前 `engine/`、`web/` 与实际剧本输出
2. 当前测试所锁定的行为
3. 现行 `handbook/`
4. 这份 `docs/` 下的当前入口文档

---

## 9. 用这份摘要检查一致性时，重点看什么

### 问卷

- 是否还能产出当前主链需要的字段
- 是否没有继续强化已降级的旧 `objectives` 口径

### 剧本

- 是否仍按年末、逐年累计口径解释
- 是否没有重新出现保险目标测算
- 是否结构仍是 综合建议摘要/A/B/C/C3-C6

### 程序

- 是否仍遵守 handbook 中已明确的 fallback / routing / bucket / insurance 规则
- 是否没有重新把旧 spec 中已废弃的 A-E 结构或旧 helper 逻辑拉回主链
