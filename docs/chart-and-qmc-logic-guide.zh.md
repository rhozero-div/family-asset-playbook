# 图表、收益测算与 QMC 逻辑说明

这份文档面向顾问，专门解释剧本里的图表、收益测算逻辑，以及 `QMC` 在当前系统中的作用。

English version: [chart-and-qmc-logic-guide.md](chart-and-qmc-logic-guide.md)

目标不是让顾问学习量化或编程，而是帮助顾问回答三个问题：

1. 每张图到底在展示什么
2. 图上的区间、线、阴影是怎么来的
3. 怎样向客户解释这些图，既不失真，也不过度技术化

本文档只讲当前生产逻辑，不讲实现代码细节。

## 1. 当前图表大致分成两类

### 1.1 确定性推演

这类输出回答的是：

- 如果先不考虑投资收益，仅靠当前资产和未来现金流，重大节点能不能覆盖
- 到每个重大节点年份时，大致有多少资金可用
- 如果会出缺口，最早在哪个节点出现

这类图和表的价值，在于先把“资金够不够”这个问题单独拎出来。

### 1.2 带收益的区间推演

这类输出回答的是：

- 如果各 bucket 参与投资收益，可能的结果区间有多宽
- 家庭总资产路径在不同情景下会怎么展开
- 单个 bucket 在偏弱和偏顺利路径下会怎么表现

这部分不是收益承诺，而是用来做情景讨论的。

## 2. 主要图表怎么读

### 2.1 家庭总资产路径图

这张图通常会同时包含：

- 年度现金流入
- 年度现金流出
- 不含投资收益的现金流参考线
- 加入投资收益后的总资产区间带

对客户的解释方式可以是：

- 柱子代表家庭现金进出
- 虚线代表“不考虑投资收益时”的累计路径
- 阴影区间代表“考虑投资收益后，不同情景下资产可能走到哪里”

重要的是：

- 带收益的路径**不保证**一定高于不投资参考线
- 投资可能带来帮助，也可能在某些年份拖累结果

### 2.2 各层余额堆叠图

这张图会把所有 bucket 放在一张图里看，例如：

- 应急层
- 各个事件资金层
- 富余资金账户

解释方式可以是：

- 每个色块是一层心理账户
- 事件 bucket 会在目标年份之前逐步累积，在目标年年末支取后清零
- 外层区间带显示的是总资产在不同情景下的大致波动范围

### 2.3 各层余额与资金来源图

这组图会把每个 bucket 单独展开来看，通常包括：

- 上年滚存
- 当年现金投入
- 当年收益
- 年末区间余额

它的好处是：

- 帮客户看清楚“钱是怎么长出来的”
- 把“靠储蓄积累”和“靠市场波动”拆开
- 让客户看到某个年份即使收益为负，长期路径仍可能整体可行

## 3. 分位数区间是什么意思

当前剧本里常见的分位数有：

- `p10`
- `p25`
- `p50`
- `p75`
- `p90`

对客户解释时，通常不需要直接说这些术语，更适合说：

- 偏弱结果区间
- 常见结果区间
- 居中结果
- 偏顺利结果区间

可以把它理解为：

- `p50` 是居中结果
- `p25-p75` 是更常见的中段区间
- `p10-p90` 是更宽的外层区间

它们是分布摘要，不是承诺，也不是硬边界。

## 4. 为什么图表里看起来已经足额，账户还在继续投入

这是当前系统里一个很容易让人误解的点。

图表里看到的 `p50` 或“居中结果余额”，是一个**展示结果**，不是停止投入的控制开关。

要把这件事分成两层看：

1. 图表中的 `p50` 只代表中间结果  
   它说明在居中的那条路径上，这个心理账户在该年年末可能已经达到或超过目标金额。  
   但这不代表所有可能路径都已经足额。

2. 当前投入规则不是“`p50` 一足额就停止投入”  
   系统会按每条路径当下距离目标还差多少，结合距离事件还有多少年，继续逐年补足。  
   因此，即使图表里的居中结果已经足额，偏弱路径下仍可能存在缺口，账户就仍会继续获得一部分投入。

3. 越接近目标，新增投入通常会变少  
   当前逻辑不是无限追加，而是按“剩余缺口 / 剩余年数”逐步补齐。  
   所以视觉上常会出现：账户已经接近甚至超过目标，但投入没有立刻归零，而是逐年收缩。

4. 当前模型不会在事件发生前自动把超出的收益提前提走  
   只要事件年份还没到，账户里的本金和投资收益都会继续留在该心理账户里滚动。  
   系统默认在事件年份年末统一支取，而不是在中途看到超额后就自动转出或锁定收益。

因此，如果图表里出现“居中情景已经足额甚至超额，但账户仍有少量继续投入”，并不表示计算错误。

它表示当前模型采用的是：

- 事件前持续滚动
- 按路径缺口渐进补足
- 到事件年统一支取

而不是：

- 达到居中目标后立即停投
- 提前把超出的收益锁定并转出

## 5. 为什么要用 QMC

`QMC` 是 Quasi-Monte Carlo。

用通俗的话说：

- 系统要生成很多条可能的市场路径
- 如果只是普通随机抽样，结果会更噪、更跳
- QMC 会把抽样点铺得更均匀，让同样样本量下的结果通常更稳定

顾问需要知道的是：

- QMC 是提高模拟质量的工具，不是“知道未来”的工具
- 它能让区间估计更平滑、更稳定
- 它并不会消除模型风险

## 6. 当前收益推演到底由什么驱动

当前带收益的推演，核心上由四层假设驱动：

1. 资产大类收益假设
2. 资产大类波动率假设
3. 大类间相关性假设
4. bucket 随时间变化的阶段权重

之所以 bucket 很重要，是因为：

- 近端资金和远端资金不该按同一套波动逻辑对待
- 应急钱不该像长期富余资金那样处理
- 富余资金在当前方法论里默认是更偏长期的钱

## 7. 为什么不能只看一个平均组合

当前方法论不是把整个家庭资产都看成一个无差别资金池。
它会先拆成不同 bucket。

这会让客户更容易理解：

- 应急钱应该怎么放
- 近几年要用的钱应该怎么放
- 长期富余资金又应该怎么放

所以它更像是一套家庭规划框架，而不是一个单一组合优化器。

## 8. 为什么统一按年、按年末口径

当前系统统一采用年度、年末结算口径：

- 现金流按年汇总
- 重大事件按事件年份年末扣减
- 投资收益也按年度步长处理

这样做的好处是：

- 模型逻辑一致
- 图表更容易讲清楚
- 不会伪装成一个月度精算执行工具

## 9. 为什么总区间不能简单等于 bucket 区间之和

这一点很重要。

家庭总资产的分位数区间，是按**完整组合路径**直接算出来的，不是把各个 bucket 的 `p10` 或 `p50` 逐个相加。

原因是：

- 各 bucket 共享同一组市场路径
- 分位数本来就不能简单相加
- 如果直接把 bucket 分位数相加，会把统计意义弄错

所以如果客户问为什么总区间看起来不等于各层区间之和，答案就是：

- 因为总区间是按完整组合路径算的，这才是正确统计口径

## 10. 如何解释“有些年份投资收益为负”

当前剧本里，即使是居中结果，也可能有些年份投资收益为负。

这不是 bug，反而是更诚实的表达：

- 它提醒客户投资结果本身有不确定性
- 它避免图表暗示“投资总是比不投资更好”
- 它让剧本更像规划讨论工具，而不是销售材料

比较合适的表达方式是：

- “这条区间带里包含了某些年份市场收益为负的情形，但长期整体仍可能保持可行。”

## 11. 这些图不代表什么

这些图**不代表**：

- 保证收益
- 承诺结果
- 推荐具体产品
- 精确预测市场
- 取代法律、税务、精算或保险专业意见

它们只是建立在明确假设之上的规划和沟通工具。
