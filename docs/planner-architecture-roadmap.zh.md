# 当前架构边界

这份文档只记录 `family-asset-playbook` **当前** 的产品边界。

English version: [planner-architecture-roadmap.md](planner-architecture-roadmap.md)

## 当前主链路

当前主链路是：

1. 填写问卷
2. 生成统一 YAML 档案
3. 调用引擎
4. 输出一份最终客户剧本

这份剧本已经包含：

- 综合建议摘要
- 客户情况概览
- 资产推演
- 资产配置执行方案
- 保险配置建议

## 主路径与次路径

### 主路径

当前产品合同应理解为：

- `问卷 -> 剧本`

剧本生成后，顾问还可以按需从方案 A / 方案 B 进入保险回填重算链路，再带着新增保险方案重新生成剧本。

### 次路径

仓库里仍保留 `asset_planner/` 和 `insurance_planner/`。
它们可以继续作为内部或实验模块存在，但已经不属于主要用户路径。

## 当前各层边界

### 问卷

问卷负责：

- 收集家庭事实
- 保存默认参数
- 生成统一 YAML 档案

### 剧本

剧本负责：

- 客户可读解释
- 重大节点覆盖逻辑
- 资产分层执行顺序
- 保险结构建议

它是当前主要客户交付物。

## 保险建议现在放在哪里

保险建议已经直接并入最终剧本，而不再作为一个必须单独跳转的页面。

当前保险部分包括：

- 保障缺口摘要
- 方案 A
- 方案 B
- 逐人解释
- 相关图表
- 方案 A / 方案 B 对应的保险回填入口按钮

## 冲突时以什么为准

如果文档和代码不一致，当前优先级如下：

1. 当前问卷实际产出的 YAML
2. 当前引擎行为
3. 当前测试
4. `handbook/` 里的现行合同
