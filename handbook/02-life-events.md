# 家庭事件节点(Family Life Events)

**版本:** 0.1.0
**状态:** draft

---

## 1. 概述

家庭事件是本方法论的**核心驱动**。客户的人生不是一段均质的时光,而是由若干**关键节点**串起的序列。每个节点都会改变资产配置的约束(流动性、刚性支出、风险偏好、时间窗口)。

本章定义事件类型、字段、阶段划分机制、事件变更处理,以及事件与海外资产模块的联动规则。

---

## 2. 事件类型

| 类型 | code | 典型金额量级(CNY) | 确定性来源 | 备注 |
|---|---|---|---|---|
| 购房 | `housing` | 200 万 – 2000 万 | 城市、地段、面积 | 刚需 vs 改善影响量级 |
| 教育 | `education` | 10 万 – 300 万/年 | 学段、地域 | K12/国际/本科/研究生/留学 |
| 退休 | `retirement` | 不固定金额,按替代率反推 | 退休年龄、预期生活方式 | 结构性事件 |
| 健康 | `health` | 不固定,常作预案 | 家庭健康史 | 通常 certainty=low |
| 传承 | `legacy` | 结构性 | 家庭意愿、税务安排 | 无明确金额 |
| 其他 | `other` | — | — | 兜底类型 |

### 2.1 购房(`housing`)

- **刚需**:首次置业,通常一次性大额支出
- **改善**:换房,通常伴随旧房处置
- 字段扩展(可选):`is_first_home`(boolean)、`intended_sell_existing`(boolean)

### 2.2 教育(`education`)

按学段细分:

| 子类 | 典型金额(CNY/年) | 触发海外模块? |
|---|---|---|
| K12 公立 | 5 万 – 20 万 | 否 |
| K12 国际 | 20 万 – 50 万 | 否(但增加外汇需求) |
| 本科(国内) | 1 万 – 10 万 | 否 |
| 本科(留学) | 30 万 – 80 万 | 是 |
| 研究生(国内) | 2 万 – 10 万 | 否 |
| 研究生(留学) | 30 万 – 100 万 | 是 |

### 2.3 退休(`retirement`)

字段扩展:
- `expected_replacement_ratio`(默认 0.6):退休后年收入 / 退休前年收入
- `owner`:退休者姓名
- 此事件**不消耗资产**,而是**改变资产结构**(从积累期转向分配期)

### 2.4 健康(`health`)

通常作为**预案**存在,`certainty=low`。字段:
- `scenario`(string):如"重大手术 + 长期康复"
- 不绑定金额,改为"流动性预留 + 保险覆盖度"的剧本建议

### 2.5 传承(`legacy`)

无明确金额,字段:
- `structural_intent`(string):如"房产给子女 + 现金做家族信托"
- 通常在退休后阶段出现
- **强烈建议咨询专业律师/税务师**(详见 [`06-boundaries.md`](06-boundaries.md))

## 3. 字段定义

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | 是 | 事件 ID(本档案内唯一) |
| `type` | enum | 是 | 见 §2 类型表 |
| `description` | string | 是 | 人类可读描述 |
| `timing_year` | integer | 是 | 预期发生年份(4 位) |
| `estimated_amount` | number | 否 | 预期金额(本币),可与 estimated_range 二选一 |
| `estimated_range` | [number, number] | 否 | 金额区间 [下限, 上限] |
| `certainty` | enum | 是 | `high` / `medium` / `low` |
| `owner` | string | 否 | 归属人(谁的事件) |
| 类型扩展字段 | — | 否 | 由 `type` 决定(见 §2 各类型扩展) |

**字段验证规则:**
- `timing_year` 不得在过去(过去的已发生事件不应出现在"规划"中,应归档)
- `estimated_amount` 与 `estimated_range` 二选一,不可同时缺省
- `id` 在同一档案中唯一

## 4. 事件驱动阶段划分

家庭事件时间线将客户生命周期划分为若干**规划期**(Planning Period, P):

```
P0: 当前 → 首个事件
P1: 首个事件 → 第二个事件
...
Pn: 第 n 个事件 → 退休
Pn+1: 退休 → 终老
```

**划分算法(纯文本描述,后续阶段 2 翻译为代码):**

1. 收集所有 `events`,按 `timing_year` 升序排列
2. 当前年份 → 第 1 个事件 = P0
3. 第 i 个事件 → 第 i+1 个事件 = Pi
4. 最后 1 个事件(或退休事件)→ 终老 = 最后一个 P
5. 退休事件**不作为分界点**,而是作为 Pn 内部的**风格切换**

**当前仓库中的 helper 模块补充说明:**

- `engine.period_divider` 仍保留一个轻量 helper 版本,在缺少“终老年份”输入时,会用最后事件后 `10` 年作为合成尾段
- 该 helper 主要服务旧测试与兼容代码,不改变本章对正式方法论的定义:正式含义仍是“最后事件之后进入退休/终老阶段”

**示例(基于示例客户档案):**

| 阶段 | 区间 | 关键事件 |
|---|---|---|
| P0 | 2026 – 2029 | 当前 → 改善型购房 |
| P1 | 2029 – 2032 | 改善型购房 → 国际高中 |
| P2 | 2032 – 2035 | 国际高中 → 本科留学 |
| P3 | 2035 – 2048 | 本科留学 → 退休 |
| P4 | 2048 – 终老 | 退休后 |

## 5. 事件变更处理

事件不是"录入即永久"。客户的人生规划会变化,方法论必须支持:

### 5.1 事件插入(Insert)

新增事件,触发**全剧本重算**:
- 新事件插入阶段划分
- 每个 Pi 内的 Pareto 骨架重新生成
- 跨期衔接重新检查

### 5.2 事件更新(Update)

修改事件的字段(`timing_year`、`estimated_amount`、`certainty` 等):
- 若影响阶段划分 → 全剧本重算
- 若仅影响 Pi 内部 → 仅该 Pi 重算

### 5.3 事件删除(Delete)

删除一个事件:
- 通常意味着该事件不再发生(如取消留学计划)
- 触发全剧本重算

### 5.4 变更的边界

- **过去的事件不可修改**(应归档为已发生事实)
- 删/改/增事件需要顾问与客户共同确认
- 每次变更应记录**变更理由**(`change_log` 字段,后续阶段 2 引入)

## 6. 与海外资产模块联动

事件可以触发海外资产模块的启用。判定规则(任一为真即建议启用):

| 触发条件 | 说明 |
|---|---|
| 教育事件含"留学"或"海外" | 留学是海外资产需求的主驱动 |
| 教育事件含"国际学校"且 ≥ 5 年 | 长期外汇支出需求 |
| 退休事件有海外养老意图 | 海外退休、海外医疗 |
| 客户已持有海外资产 | `assets.overseas` 非空 |
| 家庭成员有海外身份规划 | 移民、长期居留 |

启用方式:在 `assets.overseas_module_enabled` 设为 `true`,并补充海外资产字段(详见 [`01-input-schema.md` §7](01-input-schema.md))。

**关闭方式:** 关闭后保留历史数据(`assets.overseas` 字段不清空),便于后续重新启用。

---

## 7. 不适用事件

下列"事件"虽然常见,但**不应作为本方法论的事件处理**:

- **日常消费变化**(如换车、出国旅游) → 走 `income.expectations` 或收支假设调整,由引擎统一折算到年度现金流
- **加薪 / 跳槽** → 走 `income.sources` 调整
- **小型装修** → 走一次性支出(`support.one_time_expense`)
- **股票短期波动** → 不进入事件流

判定原则:**事件必须是会显著改变资产/现金流/约束的事项**。小事走其他维度,大事走事件。
