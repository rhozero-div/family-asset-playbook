# 资产大类假设

**版本：** 0.1.0  
**状态：** draft

English version: [03-asset-assumptions.md](03-asset-assumptions.md)

---

## 1. 概述

本章定义剧本生成时使用的资产假设层，包括：

- 资产大类角色
- 默认收益 / 波动率假设
- 默认相关性假设
- 顾问可覆盖边界
- 更新与版本管理规则

重要提醒：

1. 当前数值是方法论演示和规划结构用的占位默认值
2. 这些数值**不构成投资建议**，也**不承诺未来收益**
3. 实际部署应结合真实市场数据做校准

## 2. 当前核心资产大类

当前方法论围绕四类主资产运行：

| 大类 | code | 角色 |
|---|---|---|
| 固收 | `fixed_income` | 流动性支撑与刚性支出匹配 |
| 权益 | `equity` | 长期增长 |
| 保险 / 储蓄险 | `insurance` | 保障相关资本与长期锁定资金 |
| 另类 | `alternatives` | 分散化和非核心暴露 |

当前主线就以这四类核心资产为基础。

## 3. 当前默认占位假设

当前方法论默认会使用一套占位式假设，例如：

- 固收：低收益、低波动
- 权益：更高收益、更高波动
- 储蓄型保险：低波动、较低收益
- 另类：中高波动、分散作用

这些假设是规划假设，不是产品承诺。

## 4. 相关性逻辑

当前方法论也使用默认相关性矩阵。
这么做的原因是：

- 带收益的组合推演不能假设各类资产完全独立
- bucket 级和组合级的收益区间都需要相关性背景

但这些值依然只是方法论默认值，不应被视为对真实市场联动性的精确预测。

## 5. 为什么需要这层假设

假设层存在的原因是，系统必须能：

- 估算收益区间
- 估算不确定性宽度
- 区分短钱和长钱
- 生成 bucket 图表与总资产区间图

如果没有这层假设，系统仍可做确定性资金匹配，但无法做收益区间表达。

## 6. 顾问覆盖边界

顾问可以在客户档案里覆盖部分假设，尤其是：

- 收益率
- 波动率
- 相关性
- 阶段权重

但顾问不应把这些覆盖视为隐含的选品建议。
覆盖后的数值仍应：

- 能解释
- 有记录
- 与整体规划叙事一致

## 7. 不应轻易改动的东西

以下内容应保持稳定，除非方法论本身更新：

- 四大类的角色定义
- “不同 bucket 可以使用不同阶段逻辑”这一核心思想
- 非投顾边界声明

## 8. 当前运行时回退行为

如果 handbook 解析失败，或相关字段缺失，当前运行时可能会回退到内置默认值。

这个行为属于兼容性保障，不应成为长期替代 handbook 维护的手段。

## 9. 什么时候应更新这章

当这些内容发生变化时，本章应更新：

- 方法论版本明显变化
- 收益 / 波动率默认值被重新标定
- 文档解释与当前运行时行为出现偏差
