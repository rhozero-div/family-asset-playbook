# 家庭事件节点

**版本：** 0.1.0  
**状态：** draft

English version: [02-life-events.md](02-life-events.md)

---

## 1. 概述

家庭事件是这套方法论的核心驱动。
家庭不是一段均匀流动的时间，而是一串会改变约束条件的关键节点。

每个节点都可能改变：

- 流动性需求
- 刚性支出
- 时间压力
- 实际风险偏好
- 不同资金层的角色

## 2. 当前事件类型

当前常见类型包括：

| 类型 | code | 含义 |
|---|---|---|
| 购房 | `housing` | 自住购房、改善、首付支持等 |
| 教育 | `education` | K12、国际学校、本科、留学等 |
| 退休 | `retirement` | 退休切换与退休后收支变化 |
| 健康 | `health` | 健康相关压力预案 |
| 传承 | `legacy` | 传承、继承相关安排 |
| 其他 | `other` | 以上类型之外的大额支出 |

## 3. 当前最重要的事件字段

当前最关键的字段是：

- `id`
- `type`
- `description`
- `timing_year`
- `estimated_amount`

一些 owner 或补充背景字段也可能存在，但当前规划最核心的是年份和金额。

## 4. 事件为什么重要

在当前系统中，事件会直接影响：

- 重大节点确定性推演
- bucket 创建顺序
- bucket 支取时间
- 图表默认截止年份
- 摘要层的重大节点覆盖结论

因此，事件质量会显著影响整份剧本是否有用。

## 5. 当前时点口径

当前引擎采用**年末口径**：

- 现金流按年汇总
- 重大事件在对应年份年末结算
- 收益推演也按年度步长推进

这是为了保证整条建模链一致。

## 6. 阶段概念

当前方法论会根据“距离事件还有多少年”来判断阶段。
通俗地说就是：

- 近期：很快要用的钱
- 中期：中等等待期后要用的钱
- 远期：更远一些的目标资金
- 超远期：最长期的钱

这个阶段概念会影响：

- 默认 bucket 名称
- 阶段颜色和图例
- 收益推演时的阶段权重

## 7. 事件与 bucket 的关系

在当前实现里，重大事件通常会形成独立资金 bucket。

一般顺序是：

1. 先应急层
2. 再按时间顺序创建事件 bucket
3. 全部目标之外的剩余钱进入富余资金

这意味着事件顺序不是装饰性字段，而是真正会改变资金叙事。

## 8. 事件变化为什么要重算

当一个事件被：

- 新增
- 删除
- 提前
- 推迟
- 明显放大或缩小

就应当重算，因为它会改变：

- 资金顺序
- 缺口出现时间
- 阶段划分
- 高层结论

## 9. 退休为什么是特殊事件

退休不仅仅是一笔支出或一个节点，它还是收支结构切换点。

它会同时改变：

- 收入
- 支出习惯
- 投资视野
- 富余资金账户的阶段逻辑

因此退休在当前方法论里是一个核心结构事件。

## 10. 当前不适用的范围

事件框架并不是拿来处理：

- 所有日常消费偏好
- 产品级交易动作
- 企业现金流模型
- 高复杂度信托或跨境结构

一旦超出这个层次，剧本就不应被误用为完整方案。
